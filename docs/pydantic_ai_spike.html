How AgentManager._build_runner looks now

def _build_runner(self, additional_mcp_servers: list[str]) -> PydanticAgentRunner:
    cache_key = (self._model_name, tuple(sorted(additional_mcp_servers)))
    if (cached := self._runner_cache.get(cache_key)) is not None:
        return cached

    api_key = secrets_manager.get_secret(API_KEY_ENV_VAR)
    workspace_root = Path(config_manager.workspace_path)

    mcp_servers = [
        streamable_http_local(f"http://localhost:{self._mcp_server_port}/mcp/", name="GriptapeNodes"),
    ]
    for cfg in self._lookup_mcp_configs(additional_mcp_servers):
        if (built := mcp_server_from_config(cfg["name"], cfg)) is not None:
            mcp_servers.append(built)

    runner = PydanticAgentRunner(
        model_name=self._model_name,
        api_key=api_key,
        workspace_root=workspace_root,
        storage=self._thread_storage,
        instructions=self._instructions,
        workspace_config=WorkspaceToolsetConfig(workspace_root=workspace_root),
        mcp_servers=mcp_servers,
        usage_limits=DEFAULT_AGENT_USAGE_LIMITS,
    )
    self._runner_cache[cache_key] = runner
    return runner

What got removed

  • NodesPromptImageGenerationTool and the image_tool wiring inside AgentManager — the runner does not yet expose an image-generation tool. ConfigureAgentRequest.image_generation_driver is accepted but currently ignored. Phase D restores this as a real Pydantic AI tool.
  • griptape.structures.Agent, griptape.memory.structure.ConversationMemory, the static Ruleset blocks, and the output_schema = create_model(...) dance.
  • GriptapeCloudThreadStorageDriver: dropped to keep the storage layer single-shape; can be re-added against an opaque ModelMessage blob endpoint when GTC supports it.
  • The streaming repair_json + conversation_output token parser. Pydantic AI's text deltas are real text deltas; we forward them straight through.
